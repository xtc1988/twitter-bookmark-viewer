# 認証なし実装への変更計画

## 変更内容

手動インポート方式を使用するため、Twitter OAuth認証を削除し、認証なしで動作するように変更します。

## 変更が必要なファイル

1. **データベーススキーマ**
   - RLS（Row Level Security）を無効化
   - user_idカラムを削除または固定値に変更

2. **APIルート**
   - 認証チェックを削除
   - user_idを固定値または削除

3. **フロントエンド**
   - ログインページを削除
   - 認証チェックを削除
   - 直接ダッシュボードにアクセス可能に

4. **コンポーネント**
   - DashboardLayoutの認証チェックを削除
   - NavBarのログアウトボタンを削除

## 実装方針

### オプション1: 完全に認証なし（推奨）
- RLSを無効化
- user_idカラムを削除
- すべてのデータを単一ユーザーとして扱う

### オプション2: 固定ユーザーID
- user_idを固定値（例: '00000000-0000-0000-0000-000000000000'）に設定
- RLSを無効化
- 認証チェックを削除

**推奨**: オプション1（完全に認証なし）
